# Euphony Promotional Website

A complete full-stack promotional website for your Android music streaming app.

## Stack

- Frontend: React + Vite + Tailwind CSS + Framer Motion + Three.js (@react-three/fiber)
- Backend: Node.js + Express + Multer + JSON file storage
- Styling: Dark premium UI with glassmorphism, smooth micro-interactions, responsive layout

## Quick Start

### 1) Install dependencies

From `website/`:

```bash
npm install
npm install --prefix frontend
npm install --prefix backend
```

### 2) Configure environment

Backend:

```bash
cp backend/.env.example backend/.env
```

Set strong values for:

- `ADMIN_UPLOAD_KEY`
- `ADMIN_JWT_SECRET`

Frontend:

```bash
cp frontend/.env.example frontend/.env
```

### 3) Run in development

From `website/`:

```bash
npm run dev
```

- Frontend: http://localhost:5173
- Backend: http://localhost:4000

## Build + Run Production

### Frontend build

```bash
npm run build --prefix frontend
```

### Backend start

```bash
npm run start --prefix backend
```

## Deployment

### Backend (Render or Railway)

1. Set root directory to website/backend
2. Build command: npm install
3. Start command: npm start
4. Configure environment variables:
- PORT
- FRONTEND_ORIGIN
- ADMIN_UPLOAD_KEY
- ADMIN_JWT_SECRET

### Frontend (Vercel or Netlify)

1. Set root directory to website/frontend
2. Build command: npm run build
3. Output directory: dist
4. Configure environment variables:
- VITE_API_URL (point to deployed backend /api)
- VITE_SITE_URL (public site URL, for SEO consistency)

### Post-deploy checklist

1. Confirm API health endpoint returns OK
2. Confirm APK downloads work from home CTA and releases table
3. Replace sitemap domain in frontend/public/sitemap.xml with your real domain
4. Set backend FRONTEND_ORIGIN to exact frontend domain

## API Endpoints

### Health

- `GET /api/health`

### Reviews

- `GET /api/reviews`
- `POST /api/reviews`

Payload:

```json
{
  "name": "User Name",
  "title": "Great player",
  "rating": 5,
  "comment": "Loved the playback and design"
}
```

### APK Releases

- `GET /api/apks`
- `POST /api/apks/upload` (multipart form-data, requires admin bearer token)

### Admin Auth

- `POST /api/auth/login`
- `GET /api/auth/verify`

Login payload:

```json
{
  "adminKey": "your-secret-key"
}
```

Use returned token in header for protected upload:

- `Authorization: Bearer <token>`

Fields:

- `version` (required)
- `build` (optional)
- `changelog` (required)
- `apk` (required .apk file)

Downloads are served from:

- `GET /downloads/apks/:fileName`

### Contact Messages

- `POST /api/messages`

Payload:

```json
{
  "name": "User Name",
  "email": "user@mail.com",
  "message": "Hello from your website"
}
```

## Customization Notes

- Replace screenshot placeholders in `frontend/public/screenshots/` with real app captures.
- Update brand text and developer details in `frontend/src/App.jsx`.
- If needed, switch backend storage to a database (PostgreSQL/Firebase) while keeping routes unchanged.
