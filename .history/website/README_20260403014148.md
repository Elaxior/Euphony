# Euphony Promotional Website

A complete full-stack promotional website for your Android music streaming app.

## Stack

- Frontend: React + Vite + Tailwind CSS + Framer Motion + Three.js (@react-three/fiber)
- Backend: Node.js + Express + Multer + JSON file storage
- Styling: Dark premium UI with glassmorphism, smooth micro-interactions, responsive layout

## Features Implemented

1. Landing page with branding and animated hero
2. 3D hero element (Three.js via react-three-fiber)
3. Features section
4. Screenshot carousel (replace mock assets with real screenshots)
5. Reviews section
   - fetch existing reviews
   - submit new reviews with ratings
6. Download + Releases section
   - list APK versions
   - display changelog timeline
  - public download cards with release history
7. About section
8. Contact/feedback form
9. Reusable components and responsive design
10. Loading animation and subtle micro-interactions

## Quick Start

### 1) Install dependencies

From `website/`:

```bash
npm install
npm install --prefix frontend
npm install --prefix backend
```

### 2) Configure environment

Backend:

```bash
cp backend/.env.example backend/.env
```

Frontend:

```bash
cp frontend/.env.example frontend/.env
```

### 3) Run in development

From `website/`:

```bash
npm run dev
```

- Frontend: http://localhost:5173
- Backend: http://localhost:4000

## Build + Run Production

### Frontend build

```bash
npm run build --prefix frontend
```

### Backend start

```bash
npm run start --prefix backend
```

## API Endpoints

### Health

- `GET /api/health`

### Reviews

- `GET /api/reviews`
- `POST /api/reviews`

Payload:

```json
{
  "name": "User Name",
  "title": "Great player",
  "rating": 5,
  "comment": "Loved the playback and design"
}
```

### APK Releases

- `GET /api/apks`
- `POST /api/apks/upload` (multipart form-data, optional server-side/admin usage)

Fields:

- `version` (required)
- `build` (optional)
- `changelog` (required)
- `adminKey` (required if backend `ADMIN_UPLOAD_KEY` is set)
- `apk` (required .apk file)

Downloads are served from:

- `GET /downloads/apks/:fileName`

### Contact Messages

- `POST /api/messages`

Payload:

```json
{
  "name": "User Name",
  "email": "user@mail.com",
  "message": "Hello from your website"
}
```

## Customization Notes

- Replace screenshot placeholders in `frontend/public/screenshots/` with real app captures.
- Update brand text and developer details in `frontend/src/App.jsx`.
- If needed, switch backend storage to a database (PostgreSQL/Firebase) while keeping routes unchanged.

## Performance Notes

- Vite + code splitting for Three.js and animation libraries
- Lazy-loaded 3D hero component
- Optimized static assets and responsive sections
